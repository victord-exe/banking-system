Prueba Técnica

Sistema de Banca en Línea

Go / Rust / COBOL Vite TigerBeetle Docker

Descripción General

Desarrollar un sistema de banca en línea simplificado que permita a los usuarios gestionar sus cuentas bancarias de forma segura.

Stack Tecnológico Requerido

Backend (Elegir UNO)

  • Go
  • Rust
  • COBOL (Super Plus)

Frontend

  • Vite + React/Vue/Svelte (a elección del candidato)

Base de Datos

  • TigerBeetle (OBLIGATORIO - base de datos financiera distribuida)
  • PostgreSQL o SQLite (OBLIGATORIO - para datos de usuarios y autenticación)

Infraestructura

  • Docker (OBLIGATORIO)
  • GitHub (OBLIGATORIO)

Requisitos Funcionales

1. Autenticación y Seguridad

  • Sistema de registro de usuarios
  • Login seguro
  • Autenticación mediante tokens
  • Cierre de sesión
  • (Bonus) Autenticación adicional

2. Gestión de Cuentas

  • Crear cuenta bancaria al registrar usuario
  • Visualizar saldo
  • Ver historial
  • Cada cuenta debe tener un identificador único

3. Transacciones

  • Depósito: Agregar fondos a la cuenta
  • Retiro: Retirar fondos (validar disponibilidad)
  • Transferencia: Enviar dinero a otra cuenta
    • Validar cuenta destino
    • Manejar casos especiales apropiadamente
  • Las transacciones deben quedar registradas con información relevante

4. Dashboard

  • Vista general con información relevante
  • Transacciones recientes
  • Visualizaciones útiles (opcional)

5. Chat con IA (OBLIGATORIO)

  • Integración de un modelo de IA usando Model Context Protocol (MCP)
  • El usuario debe poder interactuar mediante chat para:
    • Realizar transacciones ("Transfiere $100 a la cuenta 12345")
    • Consultar saldo ("¿Cuánto dinero tengo?")
    • Ver historial ("Muéstrame mis últimas transacciones")
    • Hacer cualquier operación bancaria mediante lenguaje natural
  • La IA debe confirmar acciones críticas antes de ejecutarlas
  • El chat debe estar visible en el dashboard o como componente accesible

Requisitos Técnicos

Backend

Arquitectura

  • API RESTful
  • Buena organización del código
  • Manejo adecuado de errores
  • Validación de datos de entrada

Endpoints Mínimos Requeridos

Autenticación: - Registro de usuario - Inicio de sesión - Cierre de sesión Cuentas: - Obtener información de cuenta - Consultar saldo Transacciones: - Realizar depósito - Realizar retiro - Realizar transferencia - Consultar historial

Docker Setup

Requisito: El proyecto completo debe funcionar ejecutando docker-compose up

Debe incluir:

  • Backend (Go/Rust/COBOL)
  • Frontend (Vite)
  • TigerBeetle
  • PostgreSQL o SQLite

Los Dockerfiles y docker-compose.yml son parte de la evaluación y deben ser creados por el candidato.

Estructura de Base de Datos

Arquitectura Dual:
  • TigerBeetle: Para todas las operaciones financieras (cuentas, transacciones, balances)
  • PostgreSQL/SQLite: Para usuarios y autenticación

TigerBeetle - Accounts (Cuentas)

TigerBeetle maneja cuentas con su modelo de contabilidad. Investiga la documentación oficial para implementar correctamente.

TigerBeetle - Transfers (Transacciones)

Las transferencias en TigerBeetle tienen una estructura específica. Consulta la documentación para entender cómo registrar transacciones apropiadamente.

PostgreSQL/SQLite - Tabla: users

Diseña una tabla de usuarios que soporte la funcionalidad requerida. Considera qué información es necesaria almacenar para la autenticación y la relación con TigerBeetle.

Campos sugeridos (no limitativos): - Identificador único - Credenciales de acceso - Información del usuario - Referencia a cuenta de TigerBeetle - Timestamps
Conceptos Importantes de TigerBeetle:
  1. Sistema de contabilidad: Investiga cómo funciona el modelo contable
  2. Consistencia: Las operaciones deben mantener integridad
  3. Auditoría: Todo debe quedar registrado apropiadamente
  4. Operaciones especiales: Algunas operaciones requieren manejo particular

Recursos de Ayuda

Model Context Protocol (MCP)

  • SDK oficial Go: https://github.com/modelcontextprotocol/go-sdk
  • Documentación MCP: https://modelcontextprotocol.io
  • Modelos de IA: Puedes usar OpenRouter (https://openrouter.ai) para acceder a Claude, GPT y otros modelos

TigerBeetle

  • Documentación oficial: https://docs.tigerbeetle.com/
  • GitHub: https://github.com/tigerbeetle/tigerbeetle
  • Cliente Go: https://github.com/tigerbeetle/tigerbeetle-go
  • Cliente Rust: https://crates.io/crates/tigerbeetle

Go

  • Gorilla Mux / Chi / Gin para routing
  • GORM / sqlx para PostgreSQL (usuarios)
  • jwt-go para JWT
  • tigerbeetle-go para operaciones financieras

Rust

  • Actix-web / Rocket para web server
  • Diesel / SQLx para PostgreSQL (usuarios)
  • jsonwebtoken para JWT
  • tigerbeetle crate para operaciones financieras

COBOL

  • GnuCOBOL para compilación
  • COBOL-JSON para JSON parsing
  • Integración con PostgreSQL via ODBC
  • Integración con TigerBeetle via FFI/C wrapper
